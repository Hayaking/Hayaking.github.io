# 1.1 信息与数据

## 信息特征

1. 真实性
2. 时效性
3. 能够在空间和时间上被传递
4. 能以一定形式表示。
## 什么是数据?

数据是用来记录信息的可识别的符号,是符号的信息,是信息的具体表现形式.

数据可以是连续的值，比如声音、图像，称为模拟数据。也可以是离散的，如符号、文字，称为数字数据。
在计算机系统中，数据以二进制信息单元0,1的形式表示。

## 数据的特征

1. 数据有型和值之分
2. 数据受数据类型和取值范围的约束
3. 数据有定性表示和定量表示之分
4. 数据应具有载体和多种表现形式
## 数据和信息的关系

数据是信息的符号表示或载体,信息则是数据的内涵,是对数据的语义解释.

一般来说,从信息转换成为数据需要进行特征抽取,而从数据还原为信息需要经过数据语义解释.

不严格的场合下,对信息和数据不严格区分



# 1.2 数据库

## 什么是数据库？

数据库是相互关联的数据的集合,是存放数据的仓库.

## 数据库特征

1. 数据库按一定的数据模型组织（关系型、文档型）、描述和存储的；

2. 长期存储；

3. 可为多用户共享；

4. 数据独立性较高；

5. 统一管理。

## 数据库中的数据的特征

1. 数据的共享性
2. 数据的独立性.用户的应用程序与数据的逻辑组织和物理存储方式均无关.
3. 数据的完整性
4. 数据冗余小企鹅易扩充

# 1.3 数据库管理系统（DBMS）

## 什么是数据库管理系统

数据库管理系统（DataBaseManagementSystem，简称DBMS）是能够为数据库提供数据的定义\建立\维护\查询和统计等功能的位于用户和操作系统之间的一层数据管理软件

## DBMS的主要功能

1. 数据定义：可定义数据库中的数据对象


2. 数据操纵： 可对数据库表进行基本操作，如插入、删除、修改、查询等。
3. 数据的完整性检查功能：保证用户输入的数据满足相应的约束条件。
4. 数据库的安全保护功能：保证只能有具有权限的用户才能访问数据库中的数据。
5. 数据库的并发控制功能：是多个应用程序可在同一种时刻并发地访问数据库中的数据。
6. 数据库系统的故障恢复功能：是数据库在运行出现故障时进行数据库恢复，以保证数据库可靠运行。
7. 在网络环境下访问数据库功能。
8. 提供方便、有效地存取数据库信息的接口和工具。

## DBMS数据子语言

1. 数据定义语言（DDL）
2. 数据操纵语言（DML）
3. 数据控制语言（DCL）

# 1.6数据库系统的结构

## 1.6.1数据库的三级模式结构

三级模式是指逻辑模式,外模式,内模式.

### 逻辑模式的作用

对数据库中的数据整体逻辑结构和特征的描述,减少数据冗余,实现数据共享.一个数据库系统只能有一个逻辑模式.

### 外模式作用

使用户看不到,隔离那些无关的数据,简化用户操作,保障数据库数据安全,保证数据独立性和共享性.外模式可以有多个

### 内模式的作用

数据在数据库内部表示方式



1. 数据模型用数据描述语言给出的精确描述成为数据模式。

2. 数据模式是数据库的框架。

3. 数据库的数据模式有外模式、模式和内模式三级模式组成。

## 1.6.2映像

### 外模式---模式的映像及作用

外模式与模式之间的映像，定义并保证了外模式与数据模式之间的对应关系。两者之间的映像定义通常保存在外模式中。

当模式改变时，数据库管理员只需修改映像，而不用去修改外模式。

显然，外模式与内模式之间的映像不仅建立了用户数据库与逻辑数据库之间的对应关系，同时保证了数据的独立性，使得用户能够只按照外模式进行程序开发。

### 模式---内模式的映像及作用

模式与内模式之间的映像，定义并保证了数据的逻辑模式与内模式之间的对应关系。它说数据的记录、数据项在计算机内部是如何组织和表示的。

当数据库的存储结构改变时，数据库管理员只需修改映像，而不用修改数据模式。

由于用户或程序是按照数据的逻辑模式使用数据的，因此只要数据模式不变，用户或程序就不用做出修改仍可以按照原来的方式使用数据，从而保证了数据的物理独立性。
