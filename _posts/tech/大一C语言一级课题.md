## 算法题5

### 1.题目要求

n个小伙伴（编号从 0 到 n-1）围坐一圈玩游戏。按照顺时针方向给 n个位置编号，从 0 到 n-1。最初，第 0 号小伙伴在第0号位置,第1号小伙伴在第1号位置，依此类推。游戏规则如下：每一轮第0号位置上的小伙伴顺时针走到第m号位置，第1号位置小伙伴走到第m+1号位置，……，以此类推，第n-m号位置上的小伙伴走到第0号位置，第n-m+1号位置上的小伙伴走到第1号位置，……，第n-1号位置上的小伙伴顺时针走到第m-1位置。现在，一共进行了10\^k轮，请问x号小伙伴最后走到了第几号位置。

**输入格式**

输入共1行，包含4个整数n、m、k、x，每两个整数之间用一个空格隔开。

**输出格式**

输出共1行，包含一个整数，表示10\^k轮后x号小伙伴所在的位置编号。

### 2.设计思路

题目很长，但问题很简单，就是个不停换座位的问题。由题得，一共进行$10^{k}$轮，每次移动m，求n个人中x号在谁的位置坐。列式得$(10^{k}*m + x)$ %n。但是忽略了一个问题$10^{k}$ 可能会很大，大到溢出int类型最大值。经过网上搜索得一简单易懂的算法解决之——快速幂取模。

首先先了解快速幂算法。快速幂：计算时，指数为偶数则把指数除2,底数平方。否则拆出一底数使指数变偶数在进行计算。例如：$3^{20} = 9^{10} = 81^{5} = 81*81^{4} = 81*6561^{2} = 81*43046721 = 3486784401$

快速幂取模则是在其基础上与%运算结合诞生的产物。

例如：$3^{20}\% 4 = (9\% 4)^{10}\% 4 = 5^{10}\% 4 = (25\% 4)^{5}\% 4 = 1^{5}\% 4 \Rightarrow result = (1*1)\% 4 = 1$

## 算法题9

### 1.题目要求

约瑟夫（Joseph）问题的一种描述是：编号为1,2,…,n的n个人按顺时针方向围坐一圈，每人持有一个密码（正整数）。一开始任选一个正整数作为报数上限值m，从第一个人开始按顺时针方向自1开始顺序报数，报到m时停止报数。报m的人出列，将他的密码作为新的m值，从他在顺时针方向上的下一个人开始重新从1报数，如此下去，直至所有人全部出列为止。试设计一个程序求出出列顺序。

### 2.设计思路

模拟题目的步骤转圈移除即可,用链表来实现。

    typedef struct man{
        int num;//编号
        int pw;//密码
        struct man *pNext;
    }MAN;
    void Creatlist();//创建链表
    MAN *pop(MAN *p);//模仿python的pop函数，移除指定节点

最后就是宾馆管理系统了，我起初图方便想用WinAPI提供的ini操作函数代替文件操作来把数据按一定结构和格式存储到文件里，一来方便读写，二来结构清晰。不过系统写到中期才发现用WinAPI是给自己挖了一个大坑，因为往WinAPI函数传字符串参数必须是宽字节unicode编码(TCHAR或wchar\_t )，不知道是不是用vs的原因，但是用TCHAR是真的麻烦，对TCHAR操作还有另一套函数，虽然功能都与普通的字符串操作函数一样，但名字不一样，以至于我经常记错，所以经常记不起这些函数名字导致处处碰壁。但我也是没办法啊，我总不能把这些代码推翻重写，重写也要花大把时间的，无奈只好硬着头皮写了下去。虽然这导致了部分代码看起来比较乱。

这次写宾馆系统也是我第一次使用C语言多文件和WinAPI编程的程序，也是我迄今为止写过代码行最长的程序。虽然一些功能的实现看起来比较幼稚，但都还能用，也算是实现了，可以说是比较完美地完成了任务吧。

学习C语言本身是一件充满挑战，充满乐趣的事情，，(感觉是的，每次解决一道问题就感觉很有成就感，当发现里面的一些小技巧，小玩意就感到很多乐趣，例如猜数字，算椰子，小学生管理系统）然而，它常常会给人以枯燥乏味的感觉，（也的确如此，当自己整天用来解决一道题却毫无收获，难免有些乏味失落，看到陌生的代码，也难免不觉得乏味，特别是一些重复的代码，循环，眼花缭乱晕死人），还有就是写C语言就是在不停的写轮子，好多东西C语言都没，都要自己慢慢写，即便是一个看似简单的函数功能。

C语言知识很少就那么多，关键是将算法，思路，格式的熟练运用。C语言就包含数据类型，表达式，输入输出，选择，循环，嵌套，自定义函数，系统自带函数，数组，指针，字符串，结构体，共同体，文本操作，等这么多有限的知识。

为何它能够用到生活中的各处，那就是算法的灵活运用，思维的创新。

C语言就是一个“信任程序员”的语言，没有格式的要求，没有限制，全靠程序员严格的规范，正确的格式，以及类型的严格转换。

在学习了C语言这么长时间后，我认识到编程的严谨性，还认识到计算机只是运算速度快，它其实还是很笨的，只能保证会根据编程者编写的程序运行，不会越界，这就更要求编辑者的算法，程序严谨正确，这同时也给了编辑者很大的压力。

程序编辑过程要求一丝不苟，小到一个标点符号出错，大到一个结构体出错，都会导致程序出错。但是计算机不会找错，只会运行。

我立志成为一名优秀的程序员，学习使我快乐，学习使我满足。

吃得苦中苦，方为人上人。趁年轻，多敲几行代码吧。
